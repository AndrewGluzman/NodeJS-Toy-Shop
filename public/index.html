Routes

All HTTP methods are supported. You can use http or https for your requests.

  • https://toys4sale.herokuapp.com/toys?page=1 (GET)- returns 10 entries of toys for 1 page of entries
    • Client may limit number of entries by seting it after /limit/"yourqueryhere"?page=1. e.g 5 entries: https://toys4sale.herokuapp.com/toys/limit/5?page=1this way client may get 5 entrie on each page.
  • https://toys4sale.herokuapp.com/toys/s/?q="yourqueryhere" (GET)- Gives an option to search specific toy by name and description
  • https://toys4sale.herokuapp.com/toys/cat/"categoryname" (GET)- Gives an option to look for toys in specific toy category. There is three options girls, boys and sport.
  • https://toys4sale.herokuapp.com/toys/prices/?min=0&max=1000 (GET)- Gives an option to look for toys with specific toy price range. client must give parameters to min and max properties.
  • In order to add new toy client need send (POST) request to main adress https://toys4sale.herokuapp.com/toys/ as following example { "name": "smartphone", "info": "apple", "category": "girls", "img_url": "https://images.pexels.com/photos/187329/pexels-photo-187329.jpeg", "price": 900 } Authotication token needed in order to fulfill addition request. Client must include the authotication token in header inside the request.
  • In order to update existing toy client need send (PUT) request to main adress providing the id of the item you need to update https://toys4sale.herokuapp.com/toys/"itemforupdateID" as following with the property changed: example { "name": "tv", "info": "sony", "category": "girls", "img_url": "https://images.pexels.com/photos/187329/pexels-photo-187329.jpeg", "price": 900 } Authotication token needed in order to fulfill update request. Client must include the authotication token in header inside the request.
  • For Deletition of a toy client must provide id of the item as follows https://toys4sale.herokuapp.com/toys/"itemforDELETEID" Authotication token needed in order to fulfill update request. Client must include the authotication token in header inside the request.
